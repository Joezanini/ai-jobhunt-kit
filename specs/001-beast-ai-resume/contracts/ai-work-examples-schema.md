# Contract: AI Work Examples Annex Schema

**Version**: 1.0.0 | **Feature**: `001-beast-ai-resume`

## Purpose

Defines structure for `ai_work_examples_beast.md`, submitted alongside the resume per Beast application instructions.

## Document Structure

```markdown
# AI Work Examples — Beast Industries Application

Brief note: [1 sentence on how to read these examples]

## Example 1: [Title]

**What it is:** [2–4 sentences]

**Link / reference:** [URL or repo path]

**Why it matters for this role:** [1–2 sentences]

## Example 2: [Title]

**What it is:** [2–4 sentences]

**Link / reference:** [URL or repo path]

**Why it matters for this role:** [1–2 sentences]
```

## Required Content

| Example | Required topic | Reference type |
|---------|----------------|----------------|
| 1 | Webex Python SDK stewardship and/or technical content (blogs, talks) | Public URL (GitHub, blog, video) |
| 2 | Spec Kit / agentic tooling from JOBHUNT repository | Local repo path and/or public link if available |

## Constraints

- Exactly 2 examples (C-A01)
- Each `Link / reference` must be verifiable before submission (C-A02)
- No claims of work not performed by Joe (C-A03)
- Total length ≤ 1 page when rendered (C-A04)

## Acceptance

Document conforms when C-A01 through C-A04 pass and FR-012 / SC-006 are satisfied.
